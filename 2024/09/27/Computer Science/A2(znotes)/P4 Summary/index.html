1. Sorting and Searching Algorithms

1.1. Linear Search

How does it work?

  • The user is asked to enter an item they want to find in an array.
  • All elements of an array are searched one by one until the item is found.
  • When the item is found, the algorithm outputs a message saying the item is found and at which index/location it is.
  • If the particular item is not found, the algorithm outputs a message saying it is not found.

Time complexity: O(n).

1.2. Binary Search

Prerequisite: The list/array must be ordered.

How does it work?

  • The middle item of the list is found.
  • The middle item is compared to the user’s input.
  • If the middle item matches the user’s input, the algorithm outputs “item found.”
  • If the item is greater than the input, all elements below the middle index are discarded.
  • If the item is smaller than the input, all elements above the middle index are discarded.
  • The steps are repeated until the item is found or the list is exhausted.

Time complexity: \(O(log n)\) (base 2).

1.3. Bubble Sort

How does it work?

  • Bubble sort compares adjacent elements in the list.
  • They are swapped if the elements are in the wrong order.
  • The algorithm iterates through the array multiple times.
  • On each pass, the largest unsorted element moves to its correct position at the end.
  • The process continues until the entire array is sorted.

Time complexity (worst case): \(O(n^2)\).

1.4. Insertion Sort

How does it work?

  • The algorithm assumes the first element in the array is already sorted.
  • It compares the next element with the sorted portion.
  • If the next element is smaller, larger elements are shifted to the right until the correct position is found.
  • This process continues until the entire array is sorted.

Time complexity (worst case): \(O(n^2)\).

3. Recursion

3.1. Essential Features

  • Must have a base case (stopping condition).
  • Must have a general case, which calls itself.
  • The general case should move toward the base case.
  • Once the base case is reached, the recursion "unwinds."

4. Object-Oriented Programming (OOP)

4.1. Key Terms & Definitions

  • Objects: Instances of classes representing real-world entities.
  • Properties/Attributes: The data items/characteristics defined in a class.
  • Methods: Procedures/functions within a class.
  • Classes: Blueprints for creating objects.
  • Inheritance: A mechanism for creating a new class based on an existing class.
  • Polymorphism: The ability to use different classes through a common interface.
  • Encapsulation: Hiding the internal details of a class.
  • Getters and Setters: Methods for accessing and modifying object attributes.
